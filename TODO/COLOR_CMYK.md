# TODO: 색상 시스템 - CMYK 인쇄 대응

**배경**: MathCut은 교과서/교재 인쇄 출력이 목적이므로 화면(RGB)과 인쇄(CMYK) 색상 차이를 반드시 고려해야 함.

---

## 핵심 문제

| 항목 | 현재 | 인쇄 시 문제 |
|------|------|-------------|
| 색상 모델 | RGB/HEX (화면용) | CMYK 변환 시 채도 손실 (특히 밝은 파랑, 형광색) |
| 내보내기 | PNG / SVG | PNG는 CMYK 불가, SVG도 CMYK 미지원 |
| 검정 | `#333` `#111` | 인쇄 시 Rich Black vs Pure Black 구분 필요 |
| 흰색 배경 | `#fff` | CMYK (0,0,0,0) = 용지색, 문제없음 |

---

## 고려사항 목록

### 1. 색상 팔레트를 CMYK-safe로 정의
- 자유 color picker 대신 **인쇄 검증된 고정 팔레트** 제공 권장
- 각 색상을 CMYK 값 기준으로 설계하고, RGB는 디스플레이용 변환값으로 사용
- 예시:
  ```
  교과서 파랑:  C:80 M:40 Y:0  K:0  → RGB #3399FF
  교과서 빨강:  C:0  M:90 Y:80 K:0  → RGB #FF1A33
  교과서 초록:  C:70 M:0  Y:80 K:0  → RGB #4DCC33
  교과서 검정:  C:0  M:0  Y:0  K:100 → RGB #000000 (Pure Black)
  ```
- 현재 `STYLE_COLORS`를 이 팔레트 기반으로 교체 필요

### 2. 선(stroke) vs 채우기(fill) 색상 분리
- 현재: stroke/fill이 같은 스타일 preset에 묶여있음
- 인쇄 교재 기준:
  - **선**: Pure Black (K=100) 또는 진한 단색 권장
  - **채우기**: 연한 색(10~30% tint) 권장 — 너무 진하면 텍스트 가독성 저하
- 우클릭 색상 변경에서 선/채우기 각각 팔레트 제공

### 3. 내보내기 형식 개선 (중요)
- **현재**: PNG, SVG → 둘 다 CMYK 미지원
- **권장 추가**:
  - `PDF` 내보내기: jsPDF 라이브러리로 벡터 PDF 생성 가능 (CMYK 색상 지정 가능)
  - SVG는 색상 메타 주석(`<!-- CMYK: C:80 M:40 Y:0 K:0 -->`)을 포함해 후처리 가능하게
- **단기**: PNG 300dpi 이상 출력 옵션 추가 (인쇄 최소 해상도)

### 4. 화면-인쇄 색상 미리보기
- "인쇄 미리보기 모드" 토글: CMYK → RGB 역변환 색상으로 렌더링
  - 화면에서 인쇄했을 때 어떻게 보일지 시뮬레이션
- 라이브러리: `color` (npm) — RGB↔CMYK 변환 지원

### 5. 색상 경고 표시
- 팔레트에서 CMYK 재현 불가 색상(Out-of-Gamut) 선택 시 경고 아이콘 표시
- 예: 형광 계열 (`#00FF00` 등)은 인쇄 불가 명시

## 참고

- 교과서 인쇄 표준: 일반적으로 **CMYK 4도 인쇄**, 별색(Spot Color) 없음
- 선 두께: 인쇄 최소 0.25pt (픽셀 기준 약 0.35px @ 72dpi) — 현재 strokeWidth 2 이상이므로 문제없음
- 흰 배경 + 검정 선 조합이 인쇄 품질 가장 안정적
